# AI 审查提示词模板

下面这些模板默认用于：**AI 审查、找问题、给边界、帮复盘**。  
不用于：让 AI 直接写完整项目。

---

## 模板 1：只审不写

```text
我在做一个小练手项目。请只做代码审查，不要重写整个项目，也不要直接给完整实现。

请重点帮我看：
1. 核心状态是否清楚
2. 哪些函数职责过重
3. 哪些输入没有校验
4. 哪些边界情况没处理
5. 哪些地方未来最难维护

如果你要给建议，请用“问题 + 为什么是问题 + 一种可能的修改方向”的格式回答。
```

---

## 模板 2：拆第一版目标

```text
我有一个练手项目，请不要给完整代码，只帮我把它拆成“第一版最小可运行”。

项目描述：
[在这里写项目描述]

请只回答：
1. 第一版必须支持什么
2. 第一版不要支持什么
3. 第一版最少需要哪些状态
4. 第一版 main 函数大概会经历哪几个步骤
```

---

## 模板 3：边界情况提醒

```text
我已经自己写了一版。请不要重写代码，只帮我列出：

1. 3 个正常路径测试
2. 3 个错误输入测试
3. 3 个边界情况测试

项目描述：
[这里写项目描述或贴代码]
```

---

## 模板 4：局部 bug 分析

```text
不要直接给完整实现，只帮我分析这个 bug 可能来自哪里。

背景：
[项目描述]

代码：
[贴局部代码]

现象：
[描述现象]

请只回答：
1. 最可能的 3 个原因
2. 每个原因该怎么验证
3. 我应该先打印 / 检查哪些状态
```

---

## 模板 5：命名与职责审查

```text
请不要重写整个项目，只帮我审查命名和职责划分。

代码：
[贴代码]

请重点看：
1. 哪些类型命名让职责不清
2. 哪些函数名不像动作
3. 哪些变量名太模糊
4. 哪些函数做了不止一件事
```

---

## 模板 6：允许生成文本素材

```text
请帮我生成下面项目需要的文本素材，但不要修改我的逻辑结构，也不要设计新的系统。

项目类型：
[例如：文字版任务 / 对话状态机]

我需要：
1. 5 段 NPC 对话
2. 10 条道具描述
3. 3 条任务说明
4. 5 条错误输入提示语

风格要求：
[例如：轻松、古风、赛博朋克]
```

---

## 模板 7：复盘辅助

```text
请不要给代码，只帮我复盘我这个练手项目。

项目描述：
[这里写项目]

我现在的实现方式：
[这里写自己的思路]

请帮我回答：
1. 这次练到的核心能力是什么
2. 哪一部分最脆弱
3. 如果做第二版，应该优先改哪一处
4. 哪些地方说明我还没真正理解
```
